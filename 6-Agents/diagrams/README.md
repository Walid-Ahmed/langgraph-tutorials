# Agent Diagrams

Auto-generated graph plot for the runnable examples in [`6-Agents/`](../README.md).

Like the [workflow diagrams](../../5-Workflows/diagrams/README.md), this image is produced by the shared [`plot_graph()`](../../util.py) helper when you run the source script — a build artifact, not hand-drawn. It shows the **compiled topology**; the [tutorial README](../README.md) carries a hand-drawn sequence diagram showing the loop *executing over time*.

| `00_tool_calling_agent_graph.png` | [`../01_tool_calling_agent.py`](../01_tool_calling_agent.py) |

Two things about this single file that are easy to trip on:

- **The name and the source number don't match.** The file is `00_...`, but it's generated by `01_tool_calling_agent.py` (that script hard-codes the `diagrams/00_...png` output path). The *simple* example, [`00_tool_calling_agent_simple.py`](../00_tool_calling_agent_simple.py), calls `plot_graph()` with no path, so it writes `graph.png` into the working directory (git-ignored) rather than here.
- **One diagram covers both examples.** The simple and full agents compile to the *same* two-node shape — `llm ⇄ tools` with a conditional exit to `END`. So even though only one PNG is committed, it faithfully represents the structure of both scripts. What differs between them is the tool node's *implementation* (hand-written vs. prebuilt `ToolNode`), not the graph's wiring.
